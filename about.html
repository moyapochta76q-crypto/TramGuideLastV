О проекте TramGuide

TramGuide — это некоммерческий энциклопедический проект, посвящённый трамвайным системам мира и подвижному составу. Цель проекта — собрать в одном месте структурированную, проверенную и удобную для навигации информацию о городском трамвайном транспорте.

Что уже реализовано

  • Каталог трамвайных систем с основными характеристиками;
  • Карточки моделей трамвайных вагонов с техническими данными;
  • Интерактивная карта с выделением стран, где есть трамвайные системы;
  • Фильтрация по странам, городам, производителям и типам вагонов;
  • Адаптивный интерфейс для мобильных устройств, планшетов и десктопов.

Технологии

Сайт работает как статический проект на GitHub Pages и использует:

  • HTML5 и семантическую вёрстку;
  • CSS3 с использованием переменных и адаптивной сетки;
  • Vanilla JavaScript без тяжёлых фреймворков;
  • JSON-файлы для хранения данных о системах и подвижном составе;
  • SVG для интерактивной карты мира;
  • SimpleLightbox для удобного просмотра фотографий.

Принципы проекта

  • Открытость данных. Структура данных в JSON максимально проста и пригодна для расширения.
  • Доступность. Сайт учитывает базовые требования WCAG: читаемые шрифты, контраст, ARIA-атрибуты.
  • Производительность. Используется отложенная загрузка изображений и кеширование данных в localStorage.
  • Модульность. Логика разделена по модулям: навигация, карта, фильтры, загрузка данных.

Как помочь проекту

Вы можете помочь развитию TramGuide, предложив:

  • исправления и дополнения в данных о системах и вагонах;
  • фотографии трамваев (с указанием авторства и лицензии);
  • переводы описаний на другие языки;
  • улучшения интерфейса и кода.

Для предложений и обратной связи пишите на: info@tramguide.example.

Планы на будущее

В перспективе планируется:

  • подключение системы комментариев (например, Giscus на базе GitHub);
  • использование GitHub Discussions как форума сообщества;
  • создание Telegram-чата для оперативного общения;
  • разработка простой админ-панели для добавления и редактирования контента.

Эти функции пока не реализованы намеренно, чтобы сохранить простоту архитектуры и облегчить развёртывание на GitHub Pages.
